API for managing interesting places in cities

Data Structure

Field Type Description Required
id integer Auto-generated unique identifier No (auto)
name string Name of the place (max 255 characters) Yes
description string Description of the place No
category string Category of the place Yes
address string Street address Yes
city string City name (max 100 characters) Yes
rating number Rating between 0-5 No
submitted_by string Identifier of the user who submitted the place No (default: 'anonymous')
created_at timestamp Auto-generated creation timestamp No (auto)
updated_at timestamp Auto-generated update timestamp No (auto)

Valid Categories

cafe, restaurant, museum, park, landmark, shop, hotel, theater, bar, bakery, library, gallery, monument, mall, market

GET /api/places

Get all places with optional filtering and sorting

Query Parameters:

  • category - Filter by category (e.g., cafe, museum)
  • city - Filter by city (e.g., Paris, New York)
  • rating - Returns places with rating equal to or higher than this value (e.g., 4.0)
  • submitted_by - Filter by submitter (e.g., john_doe)
  • sort - Sort field (e.g., name, rating, created_at, submitted_by)
  • order - Sort order (ASC or DESC)

Request Examples:

GET /api/places?category=cafe&city=Paris&sort=rating&order=DESC
GET /api/places?rating=4.5
GET /api/places?submitted_by=john_doe
GET /api/places?sort=name&order=ASC

POST /api/places

Create a new place

Request Body (JSON):

{
    "name": "Example Cafe",
    "description": "A cozy place for coffee and pastries",
    "category": "cafe",
    "address": "789 Sample Avenue",
    "city": "Example City",
    "rating": 4.2,
    "submitted_by": "john_doe"
}

Response Example (201 Created):

{
    "status": "success",
    "message": "Place created successfully",
    "data": {
        "id": 150,
        "name": "Example Cafe",
        "description": "A cozy place for coffee and pastries",
        "category": "cafe",
        "address": "789 Sample Avenue",
        "city": "Example City",
        "rating": 4.2,
        "submitted_by": "john_doe",
        "created_at": "2024-01-16 09:15:00",
        "updated_at": "2024-01-16 09:15:00"
    }
}

Error Response Example (422 Validation Error):

{
    "status": "error",
    "message": "Validation failed",
    "errors": [
        "Name is required",
        "Category is required"
    ]
}

Required Fields:

  • name (string)
  • category (string)
  • address (string)
  • city (string)

Optional Fields:

  • description (string)
  • rating (number between 0-5)
  • submitted_by (string, defaults to 'anonymous')

Update an existing place

Request Body (JSON):

DELETE /api/places/{id}

Delete a place

Request Example:

DELETE /api/places/101

Response Example (200 OK):

{
    "status": "success",
    "message": "Place deleted successfully"
}

Validation Rules

  • name: Required, max 255 characters
  • category: Required, must be one of predefined categories
  • address: Required
  • city: Required, max 100 characters
  • rating: Optional, must be between 0 and 5
  • submitted_by: Optional, will default to 'anonymous' if not provided

Testing Tools

You can test this API using:

  • Postman - API testing tool
  • curl - Command line tool
  • JavaScript fetch - Browser API

curl Examples:

# GET all places
        curl -X GET http://localhost:8080/api/places

        # POST new place  
        curl -X POST http://localhost:8080/api/places \
        -H "Content-Type: application/json" \
        -d '{"name":"Test Place","category":"cafe","address":"123 Test St","city":"Test City","submitted_by":"test_user"}'

        # DELETE place
        curl -X DELETE http://localhost:8080/api/places/1
